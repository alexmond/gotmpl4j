= Getting started

== Add the dependency

For the engine plus the Sprig functions:

[source,xml,subs="attributes"]
----
<dependency>
    <groupId>org.alexmond</groupId>
    <artifactId>gotmpl4j-sprig</artifactId>
    <version>{page-component-version}</version>
</dependency>
----

The Sprig module depends on `gotmpl4j-core` transitively and registers its functions
automatically through `ServiceLoader`. If you want the engine with Go built-ins only,
depend on `gotmpl4j-core` instead.

== Render a template

[source,java]
----
import org.alexmond.gotmpl4j.GoTemplate;
import java.io.StringWriter;
import java.util.Map;

var tpl = new GoTemplate();                 // auto-loads Sprig from the classpath
tpl.parse("greet", "Hello {{ .name | upper }}!");

var out = new StringWriter();
tpl.execute("greet", Map.of("name", "world"), out);
//  -> "Hello WORLD!"
----

See the {api-core}/{page-component-version}/org/alexmond/gotmpl4j/GoTemplate.html[`GoTemplate`^] API
for every method, and {api-core}/{page-component-version}/org/alexmond/gotmpl4j/MissingKeyMode.html[`MissingKeyMode`^]
for the `missingkey` option below.

`render` is a convenience that returns the string directly:

[source,java]
----
String s = new GoTemplate()
        .parse("greet", "Hello {{ .name | title }}!")
        .render("greet", Map.of("name", "world"));   // "Hello World!"
----

== Data binding

The "dot" (`.`) is the current data value. Fields resolve against `Map` keys, JavaBean
getters, and public fields:

[source,java]
----
record User(String name, int age) {}

new GoTemplate()
        .parse("u", "{{ .name }} is {{ .age }}")
        .render("u", new User("Ada", 36));   // "Ada is 36"
----

== Controlling function loading

By default `new GoTemplate()` discovers every `FunctionProvider` on the classpath. Use the
builder for finer control:

[source,java]
----
GoTemplate tpl = GoTemplate.builder()
        .noAutoDiscovery()                 // skip ServiceLoader discovery
        .htmlEscaping()                    // enable contextual HTML auto-escaping
        .withFunctions(Map.of("greet", args -> "hi " + args[0]))
        .build();
----

== Missing and nil values

How a nil or absent value renders in a bare action is controlled by the `missingkey`
option, mirroring Go's `Template.Option`:

[source,java]
----
new GoTemplate().parse("t", "x={{ .missing }}").render("t", Map.of());
// default  -> "x=<no value>"   (Go's text/template behavior)

new GoTemplate().option("missingkey=zero")     // empty string instead of <no value>
        .parse("t", "x={{ .missing }}").render("t", Map.of());
// -> "x="                       (Helm semantics)
----

[cols="2,3"]
|===
| Option | A nil/absent action renders as

| `missingkey=default` (the default) / `missingkey=invalid` | Go's `<no value>`
| `missingkey=zero` | an empty string (what Helm expects)
| `missingkey=error` | execution fails
|===

This only affects bare actions like `{{ .x }}`; the `print`/`printf` functions keep Go's
`fmt` semantics (a nil argument prints `<nil>`).

== Reusing a parsed template

Parsing is the expensive step; execution is cheap. Parse once and execute many times — a
parsed `GoTemplate` (without HTML escaping) is safe to execute concurrently. For a
pre-resolved handle to one named template, use `compiled`:

[source,java]
----
var compiled = tpl.parse("row", "{{ .id }},{{ .name }}\n").compiled("row");
for (var record : records) {
    compiled.execute(record, writer);
}
----
